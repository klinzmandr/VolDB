VolDB Release Info

Version 1.0 (uploaded March 5, 2015)
  • added rptcategorydetail.php to report menu
  • added rptvoltimesummary report to the report menu
  • fixed incorrect checking of modified fields in mainmenu.inc
  • added check and inclusion of vol time to rptmcidtimereport using new table with all historical vol time entries.
  • added functionality to track vol. courses taken. modified various pages and reports.
Version 1.8 (uploaded 8/29/14)
  • modified datautils.php and added logging when new page is loaded
  • updated usage instances of nicEditor to add 'link' and 'unlink' buttons of textarea edit.
  • centralized definition of site dependent variables into datautils.inc and modified all page instances that referred to them. Added javascript checker to prevent use of email functionality unless the system 'from; email address is defined.
  • added recipient count value to log record
  • added first and last name to rptperiodanalysis report
Version 1.7 (uploaded 8/1/14)
  • upgraded to bootstrap3-typeahead.js in voltimeentry
  • added typeahead to volinfotabbed to provide autocomplete for city, state and zip
Version 1.6 (Uploaded 6/6/14)
  • added new report rptvoldetail to allow extraction and download of detailed volunteer time records
Version 1.5 (Uploaded 5/29/14)
  • added datepicker functionality to all pages that would provide added functionality. Used bootstrap capability in order to maintain compatibility with existing css functionality.
Version 1.4 (Uploaded 4/30/14)
  • upgraded function in sendemailvolsupd that creates linkable url's embedded in the message. The old version didn't isolate and convert the text strings properly.
  • fixed sendemailvolsupd to properly quote the href string in the closing tag of message.
  • added rptvolexpeption report to the main menu
  • modified volinfotabbed to merge volunteer detail tab with main information tab.
  • changed sending mail address in sendemailvolsupd to 'info@pacificwildlifecare.org' to get around SPAM checkers.
  • added at bottom of page a comma separated list of all emails to listspecificlist suitable for copy and pasting into email clients.
  • modified index.php to log list self admin action to voldb activity log for rpt.php
  • removed security restriction of having to be a voladmin to do vol record updates.
  • fixed default date range in rptmcidtimereport to be last calendar month.
  • fixed rptmaillogviewer drop down action to use 'onchange'
  • modfied listspecificlist to drop memstatus and mctype and added notes to get extra space for phone numbers.
Version 1.3 (Uploaded 3/20/2014)
  • added ability to delete a vol time entry record on listedittimerecord
  • added function to turn submit button RED on voltimeentry form to prevent inadvertent double entry (hopefully).
  • modified sendemailvols to check for page modifications before allowing navigation to a different page.
Version 1.2 (Uploaded 3/6/2014)
  • added admpasswordupd into directory. It had been inadvertently left behind.
  • defined the mainmenu bar to be fix to the top of the window.
  • change the sort order of the time entry list/edit to be date of service (VolDate).
  • created rptperiodsummary to use for Summary Reporting for Board
  • added editor facilities to the email composition page sendemailvols
  • modified all reports with date ranges to default to start and end date of prior month.
Version 1.1 (Uploaded 2/26/2014)
  • fixed listvolcategories to correctly check on security level
  • added display of MCID to index page for vol users reference
  • fixed maillogchecker to report email addresses with <> by adding htmlentities() to page output.
  • added link to selfte to allow user to list last 10 volunteer time entries.
  • fixed sendemailvolsupd to not note email in volunteer's correspondence and added 'action=form' to url string in tag of message.
Version 1.0 (Uploaded 2/23/2014)
  • complete and test the current volunteer email utility in voldb: sendchooser.php, sendemail.php and sendemailupd.php
  • initial release of 'index.php' to allow volunteers to manage their own lists
