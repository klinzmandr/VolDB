Volunteer Time Self Entry (VTE) Spec's

Objective: provide an on-line system to allow PWC volunteers to report their own time. The system will be available on the web supporting smart phone, tablet as well as pc input from any location at any time. The VTE may EVENTUALLY provide the ability to send email notices to all registered volunteers based one or more mailing distribution lists associated with each member. (NOTE: email support may be an integrated but with its own user interface to create and send emails to volunteers.) Identification of the individual volunteer will be via use of their unique 'PWCId' assigned when the volunteer is entered into the PWC on-line membership system MbrDB. This id is referred to as an 'MCID' and consists of the first 3 characters of the volunteers last name plus 2 numeric digits usually from the street address or phone number to make the 'PWCId' unique and easy to remember. Specification notes:
  • Administration of this system is to be performed by a 'Volunteer Administrator' who would assume overall responsibilities for the accuracy of the volunteer roster in MbrDB.
    • Designation of the individual as a 'volunteer' is done by setting the 'Mbr Status' to a value of 2 (Volunteer) on initial entry of by the Volunteer Administrator when needed.
    • When a person is registered as, or becomes, a volunteer, the 'Mem Status' for that individuall is updated to reflect that.
    • When the person is no longer an active volunteer, the 'Mem Status' field is changed to reflect the new status.
  • Anyone can report volunteer time in the system - it does not require that a person be 'designated' as a volunteer. However, only those designated as 'volunteers' (i.e. 'Mem Status' = 2) will be included on certain reports generated by the volunteer administrator.
  • Initial entry of the volunteer in the the MbrDB system should be done when registration and dues requirements have been met either my the Membership or Volunteer coordinators.
Two different entry points (pages): www.pacificwildlifecare.org/vte for volunteers and www.pacificwildlifecare.org/vte/admin for the volunteer administrator Volunteer main entry page
  • allow entry of exact MCID to produce the confirmation page, or
  • allow search for all or part of MCID, last or first name (note: this option would result in a list of candidate MCID's based on the search results) and selection of one of which would produce the confirmation page.
  • confirmation page (are you ???) with a re-try optional link if not.
  • confirmation page provides entry of a date, vol time, vol mileage, pull down of volunteer 'category' and optional Notes field. Validations would include:
    • correct date format
    • numeric value for time
    • optional entry of a numeric value for mileage
    • Vol categories select list to include:
      • Center Vol
      • BBRoom Vol
      • Transporter
      • Home Rehabber
      • Office Admin
      • Phone Volunteer
      • Event/Committee
      • Other (requires entry of Notes field)
    • Notes (optional except when 'Other' is selected from category)
  • Successful entry of page will add new row to the 'volunteertime' table of MbrDB
Administrative page
  • requires login (same as for MbrDB) with admin level of 'voladmin'.
  • main menu - same as MbrDB main menu except deletion of funding, correspondence, reminders and detailed reports (leave the report menu drop down) leaving the search and update of MCID main info page plus reporting.
  • maintenance of membership information (other than funding or corresondence) information
  • add to main menu
    • maintenance of volunteer categories
    • edit/delete of any existing 'volunteertime' record by searching for name/MCID.
    • detailed reports
      • list of active volunteers
      • list of 'inactive' volunteers (anyone without an entry within the last 3 months(?)
      • time reported for a specific MCID (summarized and/or detailed by category) within a specific date range, and
      • total time and mileage report by category within a date range (note: this would allow YTD as well as monthly reporting.)
Database table 'volunteertime'
  • VolTimeID - seq. record number
  • MCID - varchar(6)
  • volname - varchar(50) - LabelName1stline from members table
  • voltime - numeric - 2 digits plus 2 decimals - range allowed from 00.01 to 12.59
  • volmileage - numeric
  • volcategory - varchar(20)
  • volnotes - varchar(128)
